1 # AiCPlayer - README
2 
3 The AiC Player is the interface between sensors emitters and
4 the virtual android machine. It is split into several executables:
5 
6 - The most important one, which we can call "player", does hardware-accelerated
7  rendering for the android UI, using the remote rendering libs (mostly vanilla)
8  from the google emulator. Because it has access to the graphical interface, it
9  also contains code required to record screenshots and videos from the graphical
10  output.
11 - The "player_audio" is a separate executable which connects to a custom port on
12  the virtual machine in order to receive a raw PCM stream of the audio output of
13  android. Its job is to copy that stream to a local ffserver instance, in order
14  to transcode it and present it elsewhere (for example a web interface).
15 - The "player_sensors"â€™ job is to dispatch protocol buffers containing sensor data
16  from an AMQP queue to the different TCP ports for those sensors on the VM.
17  Since each sensor has its dedicated AMQP queue, player_sensors does not need
18  to filter or even read the content of the protocol buffer, only to transfer it.

55 ## Obtaining the OpenGL rendering libs
56 
57 The following libs are **required** in order to get the OpenGL rendering working:
58 
59 - lib64EGL_translator.so
60 - lib64GLES_CM_translator.so
61 - lib64GLES_V2_translator.so
62 - lib64OpenglRender.so
63 
64 Because if those libs are not available, then the graphical player will
65 not run, and android will not boot fully unless it is configured to boot
66 without hardware acceleration.
67 
68 To build them, you need to apply a patch to the AOSP sources in order to
69 be able to receive rotation events, then build it using the scripts available
70 to build the emulator (which may differ between android versions).

72 # Usage
73 
74 Runtime parameters are given through environment variables, which are
75 all mandatory. Running with docker-compose is recommended (and easier)
76 in order to have all executables running at the same time with the right
77 parameters.

80
