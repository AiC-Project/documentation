00001 # AiCPlayer - README
00002 
00003 The AiC Player is the interface between sensors emitters and
00004 the virtual android machine. It is split into several executables:
00005 
00006 - The most important one, which we can call "player", does hardware-accelerated
00007   rendering for the android UI, using the remote rendering libs (mostly vanilla)
00008   from the google emulator. Because it has access to the graphical interface, it
00009   also contains code required to record screenshots and videos from the graphical
00010   output.
00011 - The "player_audio" is a separate executable which connects to a custom port on
00012   the virtual machine in order to receive a raw PCM stream of the audio output of
00013   android. Its job is to copy that stream to a local ffserver instance, in order
00014   to transcode it and present it elsewhere (for example a web interface).
00015 - The "player_sensors"’ job is to dispatch protocol buffers containing sensor data
00016   from an AMQP queue to the different TCP ports for those sensors on the VM.
00017   Since each sensor has its dedicated AMQP queue, player_sensors does not need
00018   to filter or even read the content of the protocol buffer, only to transfer it.

00055 ## Obtaining the OpenGL rendering libs
00056 
00057 The following libs are **required** in order to get the OpenGL rendering working:
00058 
00059 - lib64EGL_translator.so 
00060 - lib64GLES_CM_translator.so
00061 - lib64GLES_V2_translator.so
00062 - lib64OpenglRender.so 
00063 
00064 Because if those libs are not available, then the graphical player will
00065 not run, and android will not boot fully unless it is configured to boot
00066 without hardware acceleration.
00067 
00068 To build them, you need to apply a patch to the AOSP sources in order to
00069 be able to receive rotation events, then build it using the scripts available
00070 to build the emulator (which may differ between android versions).

00072 # Usage
00073 
00074 Runtime parameters are given through environment variables, which are
00075 all mandatory. Running with docker-compose is recommended (and easier)
00076 in order to have all executables running at the same time with the right
00077 parameters.

00079 ## Generic options
00080 
00081 Option                      | Use
00082 ---                         | ---

00091 Option                      | Use

00099 ## Sensors player options
00100 
00101 Option                      | Use
00102 ---                         | ---
00103 AIC_PLAYER_ENABLE_SENSORS   | Enable the misc sensors like accelerometer, thermometer, luxmeter…

00117 ## Remote commands
00118 
00119 Remote commands/sensor data are sent through AMQP queues; read
00120 the specific message documentation for more information.
00121 
00122 Queue name                       | Effect
00123 ---                              | --- 
00124 android-events.{vm_id}.battery   | Forward a battery payload
00125 android-events.{vm_id}.sensors   | Forward a sensors payload
00126 android-events.{vm_id}.gps       | Forward a GPS payload
00127 android-events.{vm_id}.gsm       | Forward a GSM payload
00128 android-events.{vm_id}.nfc       | Forward a NFC payload
00129 android-events.{vm_id}.recording | Toggle video recording or take a screenshot
